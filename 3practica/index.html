Daniela Monterrubio Palma
Ing. Biomédica
Fecha: Septiembre 7, 2023
Arduino Attiny85 LED Push button

- Resumen -

La práctica 3 tuvo como objetivo comenzar la programación básica de entradas y salidas digitales usando Attiny85. Esto se realizó utilizando LEDs y push buttons, por lo que fueron conexiones de componentes previamente vistas. Asimismo, el código de cada actividad ya estaba programado dentro de Tinkercad, por lo que se repasaron las instrucciones para cada componente. Sin embargo, la diferencia con las prácticas anteriores fue que el Attiny85 no requiere de una conexión permanente con el Arduino UNO para hacer funcionar el circuito. Es decir, el Attiny85 sí usa energía, pero puede valerse de cualquier fuente de energía, ya que las instrucciones para el funcionamiento del circuito se quedan grabadas en el Attiny85.

- Introducción -

Attiny85 es el microcontrolador más pequeño de Arduino. Además, no es necesario utilizar capacitores, resistores y cristal externos para su funcionamiento. Este microcontrolador cuenta con 8 pines y no tiene puerto USB, por lo que los proyectos a realizarse deben ser con una cantidad limitada de componentes. Entre sus especificaciones están su arquitectura AVR (Advanced Virtual RISC), voltaje de alimentación es 2.7 - 5.5 V, velocidad de CPU de 20 MHz y memoria RAM de 8 Kbytes [1].

El Bootloader es un código ejecutado con cada reinicio que busca un archivo a subir mediante el puerto USB. Para subir el Bootloader al microcontrolador se requiere de un ISP (In-circuit Serial Programmer), el cual se conecta a pines específicos del microcontrolador. En el caso de esta materia, el Arduino UNO es el que se usará como ISP para cargar todos los códigos al Attiny85 y que este pueda funcionar sin necesidad de conectarse a la computadora [2].

• 1 Protoboard

• 3 LEDs

• 2 Push button

• 3 Resistencias de 220 Ω

- Conclusiones -

En conclusión, el objetivo de la práctica se cumplió, dado que se comprendió el funcionamiento de las entradas y salidas digitales del Attiny85, al igual que se emplearon correctamente. En un principio hubo dificultad para comprender cómo había que conectar el Attiny85 al Arduino UNO, pero eventualmente se comprendió cada uno de los puertos. Asimismo, se observó que el Attiny85 no requería de una conexión directa a la computadora para poder ejecutar el código. Así que se aprendió en su totalidad la manera en la que había que trabajar con el microcontrolador.
